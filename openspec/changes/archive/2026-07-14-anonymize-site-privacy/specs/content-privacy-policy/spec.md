## ADDED Requirements

### Requirement: 禁止出现真实雇主名

全站**已发布内容**（`docs/**/*.md` 及站点配置中的可见文案）MUST NOT 出现可识别的真实雇主名称，MUST 以泛化的品类 / 规模描述替代。

#### Scenario: 正文不含真实雇主名

- **WHEN** 对 `docs/**/*.md` 全文检索雇主关键词（如 `360`、`腾讯`、`Tencent`、`鹅厂`）
- **THEN** 命中数为 0，且对应位置已替换为泛化描述（如"某互联网安全公司""某头部游戏大厂"）

#### Scenario: 经历骨架保留

- **WHEN** 阅读个人经历章节
- **THEN** "从互联网安全后台到游戏后台"的两段式经历骨架与技术方向仍然存在，仅去除可识别的雇主标识

### Requirement: 禁止出现具体项目 / 产品代号

全站已发布内容 MUST NOT 出现具体的项目、游戏或产品代号，MUST 以品类化描述替代，同时保留其技术场景以维持知识价值。

#### Scenario: 项目代号已泛化

- **WHEN** 检索项目 / 产品代号（如 `微视`、`天美`、`逆战`、`怪物猎人`、`星之破晓`、`宝可梦大集结`）
- **THEN** 命中数为 0，且替换为品类描述（如"某短视频中台""某头部游戏工作室""某射击手游""某端转手游上云项目""某格斗手游""某 MOBA 手游"）

#### Scenario: 技术场景不丢失

- **WHEN** 阅读脱敏后的项目相关段落
- **THEN** 原有的技术场景（上云、商业化、秒杀、限流等）与结论仍然完整可读

### Requirement: 内部专有系统名等价替换

全站已发布内容 MUST NOT 直接出现前雇主内部专有系统 / 框架的代号，MUST 替换为**通用等价物**，并在首次出现处以**公开可比对象**做类比，使读者仍能理解其技术定位。

#### Scenario: 内部系统名已替换

- **WHEN** 检索内部系统代号（如 `tcaplusdb`、`tbus`、`tconnd`、`xmesh`、`xMesh`、`tRPC`）
- **THEN** 命中数为 0，且替换为通用等价物（如 分布式 KV 存储 / 内部消息总线 / 长连接接入网关 / 自研服务网格 / 自研 RPC 框架）

#### Scenario: 首次出现带类比注解

- **WHEN** 某通用等价物在一篇文档中首次出现
- **THEN** 其后附一句公开类比（如"长连接接入网关（职能类似公有云接入层 / LVS+长连接）"），保证技术定位清晰

### Requirement: 文件 slug 与链接不泄露内部代号

站点的文档 slug、内部链接与导航配置 MUST NOT 以内部代号命名，重命名后所有引用 MUST 保持可达（无死链）。

#### Scenario: slug 已泛化

- **WHEN** 列出 `docs/**/*.md` 文件名与 `docs/.vuepress/configs/*.js` 中的 `link` 路径
- **THEN** 不含 `tconnd`、`tbus`、`xmesh`、`tcaplus` 等内部代号片段

#### Scenario: 无死链

- **WHEN** 执行 `npm run docs:build`
- **THEN** 构建成功且无因重命名导致的断链 / 404 告警，侧边栏、导航与跨文档链接全部指向新 slug

### Requirement: 移除个人身份 PII

全站已发布内容 MUST NOT 出现可定位到个人的身份线索。

#### Scenario: PII 已移除

- **WHEN** 检索个人身份线索（如 `super marco`、`marco`、`0x06` 等昵称 / 域名 / 联系方式）
- **THEN** 命中数为 0 或已替换为非个人化表述

### Requirement: 术语替换全站一致

同一实体在全站 MUST 被替换为同一泛化词，MUST 维护一份术语映射表作为一致性依据，避免半脱敏或前后不一致。

#### Scenario: 一致性可核对

- **WHEN** 抽查同一实体在不同文档中的替换结果
- **THEN** 均使用术语映射表中的同一泛化词，无混用、无残留原始代号

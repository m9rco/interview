# dns-attack-tunnel — 闪卡

> DNS 三原罪（明文UDP/全网放行/递归代理）使其既是攻击目标又是外带载体，防御靠 DNSSEC+行为统计+出网收敛。

## 记忆口诀

**三原罪**：明文UDP无认证 / 53全网放行 / 递归天然代理
**四攻击**：投毒猜ID / 反射靠放大 / 水刷缓存miss / 隧道编码子域
**隧道特征**：高熵子域 / 低缓存命中 / 异常QTYPE / 长会话
**防御链**：DNSSEC / RPZ限速 / 熵分类器 / 出网收敛

## Card 1

**Q**: UDP DNS 只靠什么来绑定"请求-响应"？为什么这成了缓存投毒的攻击面？

**A**: 只靠 16-bit 事务ID + 源端口 + 目的端口 + Question 四要素的熵。无连接无握手，若源端口固定则可猜测熵仅 16 bit（6.5万种），攻击者可抢在真实权威前伪造响应抢答。防御是端口随机化 + 0x20 大小写混淆提熵，DNSSEC 用签名根治。

## Card 2

**Q**: Kaminsky 攻击相比传统缓存投毒的"放大点"在哪里？

**A**: 传统投毒一次 miss 只有一个抢答窗口，猜错要等 TTL 过期。Kaminsky 不停查询 `随机N.bank.com`，每次必然 miss 触发向权威的查询，从而获得海量抢答窗口；一旦某次伪造响应用 Additional 区段塞进 NS/glue，就能污染整个域而非单条记录。

## Card 3

**Q**: DNS 隧道能穿透多数企业防火墙的物理基础是什么？

**A**: 递归解析的"任意子域都会被如实转发到对应权威"特性。企业出口通常放行53端口且不深检 DNS 内容；即便封直连，内网主机→内网递归器→外部权威这条合法链路依然通畅，数据被编码进子域"驮"出去。

## Card 4

**Q**: 对比：检测 DNS 隧道用"静态签名"还是"行为统计"更好？为什么？

**A**: 行为统计更鲁棒。隧道的编码字母表、分片大小可任意调整，静态签名/域名黑名单易被绕过且永远滞后。而高熵子域 + 低缓存命中 + 异常QTYPE分布 + 长会话是隧道为传输效率与穿透性付出的代价，难以同时消除，因而更难规避。

## Card 5

**Q**: 为什么"只封 53 端口"的防御不够？

**A**: 53 端口业务必需，一封全断；且隧道可走 DoH/DoT（443/853）伪装成 HTTPS 绕过纯端口策略。正确做法是收敛出网路径（只允许经受控递归器出网、封主机直连），再在递归器/出口做内容与行为层面的检测。

# graph — 闪卡

> **图论题先认清三件事：有向/无向、带权/不带权、稠密/稀疏。不带权最短路→BFS（逐层扩散）；带非负权最短路→Dijkstra（堆优化）；依赖排序/判环（DAG）→拓扑排序（Kahn 入度法）；连通性/合并集合→并查集（路径压缩 + 按秩合并近 O(1)）；遍历/连通分量/岛屿→DFS/BFS。**

## 记忆口诀

- **认图三问**：有向无向 / 带权不带权 / 稠密稀疏
- **最短路**：不带权→BFS逐层 / 非负权→Dijkstra堆 / 负权→Bellman-Ford / 全源→Floyd
- **DAG**：拓扑排序→Kahn入度0出队 / 输出<n即有环
- **连通性**：并查集→路径压缩+按秩合并→近O(α)≈O(1)

## Card 1

**Q**: 给定一张不带权图求单源最短路，为什么用 BFS 而不是 DFS？BFS 靠什么保证"首次访问即最短"？

**A**: 队列 FIFO 保证按层扩散，第 k 层节点到源距离恰为 k，故首次出队即最短。DFS 会先走深再回头，先到的不一定最近。

## Card 2

**Q**: Dijkstra 和 Bellman-Ford 都求单源最短路，二者如何取舍？

**A**: Dijkstra 贪心 + 小顶堆，O(E log V)，但要求非负权（负权破坏"最近点已定"前提）。Bellman-Ford 松弛 V-1 轮，O(VE)，可处理负权并判负环，但更慢。

## Card 3

**Q**: 判断有向图是否有环有哪两种做法？拓扑排序如何同时给出顺序？

**A**: ①拓扑排序（Kahn 入度法）：入度 0 出队、删边，若输出节点数 <n 则有环，出队顺序即拓扑序；②DFS 三色标记，遇灰色回边即有环。

## Card 4

**Q**: 并查集为什么单次操作近 O(1)？两大优化各起什么作用？

**A**: 路径压缩（find 时挂到根）+ 按秩合并（矮树挂高树）合用后摊还 O(α(n))，α 反阿克曼函数实际 ≤4。只用其一仅 O(log n)。

## Card 5

**Q**: 存图选邻接表还是邻接矩阵？各自适合什么场景？

**A**: 邻接表省空间、遍历邻居快，适合稀疏图；邻接矩阵 O(1) 查任意边、实现简单，适合稠密图或频繁查边。

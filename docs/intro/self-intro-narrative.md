# 面试开场自我介绍叙事脚本

> 主线：**风险把控 + 系统稳定性**。副线：不自夸，让技术判断力从取舍与踩坑里自然透出。
> 用法：先背熟「60 秒电梯版」；被追问就顺着「引导路线图」把话题引到我最有弹药的深水区。**按需裁剪，不要逐字背。**

::: tip 一句话定位
我这九年一直在做同一件事——**在钱和状态不能出错的系统里，把风险提前摁住，把稳定性做成可复盘的确定性**。360 三年练的是攻防和高并发的底子，鹅厂六年是把它落到支付、自研 Mesh 和万级有状态服务上。
:::

## 一、钩子开场（15 秒抓注意力）

> **可直接念：**
> "面试官好。先给您一个我这些年的判断标准：**一个系统好不好，不看它跑得多快，看它出事时你能不能睡得着觉。** 我九年后台，前三年在某互联网安全公司做 DDOS 攻防和智能硬件后台，后六年在鹅厂做游戏商业化和自研服务网格——**两段经历的共同点，都是'一旦出错就是资损或全服故障'的场景**。所以我今天想主要聊两件事:**风险怎么提前把控,稳定性怎么做成可验证的。**"

**为什么这样开场**:一句话给面试官一个记忆锚点(「出事能不能睡着觉」),同时预告主线,把后面的提问权悄悄握在自己手里。

## 二、经历骨架（90 秒，每段以「风险 / 我怎么把控」收束）

**互联网期(某互联网安全公司 · 3 年)**——智能硬件后台 + 网络安全 DDOS 攻防。
- 做过 CoreDNS/PublicDNS、Erlang 高并发、内核编译、ARM 信创;Web 全栈从 jQuery 到 Vue/React。
- **最大的风险**:DDOS 场景下流量是对抗性的,清洗规则一旦误伤就是真实用户被拦。**我怎么把控**:把清洗和拦截拆成可灰度、可回滚的两段,先观测再动手。
- ➡️ *面试官若追问网络/攻防* → 顺势引到 [DNS 攻防与隧道](../internet/dns-attack-tunnel.md)、[LVS 与 epoll](../internet/lvs-epoll.md)、[TCP 网络](../internet/tcp-net.md)。

**游戏期(某头部游戏大厂 · 6 年)**——自研 Mesh、编译优化、商业化系统、研效工具、AI 与游戏后台。
- 某射击手游**商城/充值/抽奖/VIP 系统负责人**,从 0→1 做商城与支付、付费测试到上线。
- **最大的风险**:商业化链路里全是钱,「同一个请求处理两次」就是资损。**我怎么把控**:每个改余额/库存/道具的写操作,我都先问一句「它被调两次结果一样吗」,不一样就上幂等。
- ➡️ *面试官若追问支付/商业化* → 引到 [业务代理·支付·商城](../game-biz/business-proxy.md)、[业务幂等性设计](../game-biz/idempotency-design.md)。
- ➡️ *面试官若追问架构/网格* → 引到 [自研 Mesh × K8s](../game-infra/self-mesh-k8s.md)、[Istio vs Cilium](../game-infra/mesh-istio-cilium.md)。

## 三、稳定性 / 风险信条（60 秒，主线锚点，最经得起追问）

> **可直接念：**
> "我把高可用拆成四个动作,遇事对号入座就行:**请求太多——限时、排队、削峰;下游变慢——超时、重试、熔断、隔离;核心挂了——降级、冗余、故障转移;重复请求——幂等、防重、状态机。** 上线前还有五道防线:**压测、监控、混沌、演练、容量评估。** 这套不是背出来的,是踩过坑攒出来的。"

**踩坑复盘一:支付回调双发限定皮肤(幂等)。**
- **风险画面**:回调「先查订单没发过 → 发货 → 标记已发」,MQ 至少一次、回调重投,两个协程几乎同时查到「没发过」,**各发一份**,玩家白嫖限定皮肤,对账才发现少收钱。根因是**先查后写之间有并发窗口**。
- **我怎么填**:上**四道闸**——幂等键由发起方生成且重试不变、数据库**唯一索引**做存储层原子兜底、CAS/状态机保证只生效一次、对账兜底。
- ➡️ 深挖 → [业务幂等性设计](../game-biz/idempotency-design.md)。

**踩坑复盘二:自研 Mesh 从 Consul 到全连接单跳广播(万级有状态)。**
- **风险画面**:Consul 版在规模上有硬伤(300 实例数据重复、约 5000 节点上限),节点一多就雪崩。
- **我怎么填**:改成**「一机一 Mesh + hostNetwork」跳出 K8s Overlay**,用**全连接单跳广播**换 O(1) 收敛,选连接目标用**蓄水池抽样**,路由用**一致性哈希**减少扩缩容搬迁。
- **这里我会主动澄清一句**:"我们内部管它叫 Gossip,但严格讲**代码是全连接单跳广播,不是真正的 Gossip**——没有感染轮次、没有随机邻居转发。我倾向于把概念和落地分清楚讲。" ——*这句「主动认边界」比任何自夸都显判断力。*
- ➡️ 深挖 → [自研 Mesh × K8s](../game-infra/self-mesh-k8s.md)、[蓄水池抽样](../game-infra/reservoir-sampling.md)、[一致性哈希实现](../game-infra/consistent-hash-impl.md)。

## 四、埋钩子引导（主动抛深水区，让面试官顺着我问）

> **可直接念(抛钩子):**
> "如果时间够,有三个点我特别愿意展开:**一是为什么我们没上 Istio 而是自研 Mesh;二是支付幂等的四道闸具体怎么落地、哪道是底线;三是万级节点怎么用一个算法把连接数摁下来。** 您对哪个更感兴趣?"

- **钩子 A — 为什么不用 Istio**:不是控制面慢,是每个 Pod 的 Envoy——协议栈反复横跳、L7 解析重、**XDS 全量下发**(500 服务 ×10 Pod = 每个 Envoy 维护 5000 实例,扩缩容一波爆炸)。→ [Istio vs Cilium](../game-infra/mesh-istio-cilium.md)、[eBPF](../game-infra/ebpf.md)。
- **钩子 B — 幂等四道闸的底线**:唯一索引是最硬的底线,因为原子性由数据库在存储层保证,应用层再怎么并发也插不进第二条。→ [幂等性设计](../game-biz/idempotency-design.md)。
- **钩子 C — 超时治理的渐进式演进**:点直购超时问题我们做过 PLAN A/B/C 的逐步演进,每一版都在收敛风险面。→ [业务代理·支付·商城](../game-biz/business-proxy.md)、[限流熔断](../game-infra/ratelimit-circuitbreak.md)。

**为什么埋钩子**:三个钩子并行摆出,面试官岔到任何一个我都有弹药;真被岔开了,我也能用「回到刚才那条主线」收回来。

## 五、收口反问（用反问把控节奏，体现判断力）

> **可直接念：**
> "所以我给自己的定位很简单:**在不能出错的地方,把风险前置、把稳定性做成可复盘的确定性。** 我想反问一个问题——**咱们这个岗位当前最大的稳定性风险,是在容量、在依赖治理、还是在发布环节?** 我好判断我过去哪段经验最能直接接上。"

**为什么用反问收口**:一句话把「我是什么样的工程师」立住,再用一个精准问题把话题拉回岗位真实痛点——既显得懂业务,又自然进入技术深聊。

## 60 秒电梯版（独立可念，不依赖上文）

> "面试官好。我九年后台,前三年在互联网安全公司做 DDOS 攻防和智能硬件后台,后六年在鹅厂做游戏商业化和自研服务网格。**这两段的共同点,都是一出错就是资损或全服故障的场景**,所以我最擅长的是**风险把控和系统稳定性**。
>
> 举两个我踩过的坑:一个是**支付回调重投导致限定皮肤被双发**,我用幂等键加数据库唯一索引加状态机四道闸摁住了资损;另一个是**自研 Mesh 从 Consul 迁到一机一 Mesh 加主机网络**,用全连接单跳广播和一致性哈希撑起了万级有状态直连。
>
> 我给自己的定位是:**在不能出错的地方,把风险前置、把稳定性做成可验证的确定性。** 您想先从哪块聊?"

## 3~5 分钟完整版（把五段串成连贯口述）

按顺序把上面五段口语连起来即可:**钩子(「出事能不能睡着觉」)→ 两段经历骨架(各以风险收束)→ 稳定性四板斧 + 两个踩坑复盘 → 抛三个钩子问面试官想听哪个 → 反问岗位最大稳定性风险收口。** 中间每讲完一个踩坑,停顿半秒给面试官插话的空档——**让他追问,而不是我硬灌。**
